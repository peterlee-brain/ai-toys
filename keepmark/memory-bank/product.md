# KeepMark · 留标 — 产品说明

## 一句话

**KeepMark 在你读英文网页时，用最简单的交互快速获取翻译与语法，记录你的每一次阅读行为；日积月累，你会逐渐像母语者一样读英文。**

---

## 产品是什么

KeepMark（留标）是一款 Chrome 浏览器插件，嵌入 Medium、新闻、论文等任意英文网页。

它不把你拽离文章，也不要求复杂操作：**选中即译**、**一键语法**、**点 ★ 留标**——把你在真实阅读里的翻译、讲解、标记等行为完整保存下来，形成一条随时间增长的 **个人英语阅读轨迹**。  
这些轨迹来自你真正读过的句子和页面，而不是脱离语境的单词表；长期积累后，目标是让你在英文阅读中 **更少停顿、更少查词、更多直觉理解**。

---

## 解决的痛点

| 痛点 | KeepMark 的做法 |
|------|-----------------|
| 查词跳转打断阅读流 | 选中后 **自动释义**，留在当前页 |
| 只懂词义，不懂句子怎么写、怎么读 | 按需打开 **语法讲解**，补齐结构理解 |
| 学习记录散落，无法复盘自己怎么变强的 | **保存阅读行为**（查词、语法、留标、语境） |
| 生词本太杂或太空，和真实阅读脱节 | 句内词库 + 留标机制，只放大 **你关心的词** |
| 背了词仍读不顺，缺少「像母语者一样读」的路径 | 持续在 **真实语料** 上练，靠行为数据长期复利 |

## 核心功能

### 1. 选中即译（最简单交互）

- 拖选 **单词或短语** → 自动弹出释义 Popover
- 拖选 **完整句子** → **不弹 Popover**，自动打开 Side Panel「学习」Tab 并加载语法讲解
- 无需再点「翻译」，尽量 **零额外步骤**
- 轻量 Popover，不打断阅读节奏
- 翻译请求同样携带 `selection` + 系统补全的 `sentence`（规则见「语境补全」）

**完整句子判定（与实现一致）：**

- `selection` 含空格（非单词级选中），且满足其一：
  - 以 `.` `!` `?` 结尾
  - 与系统补全的 `sentence` 文本一致（或仅差句末标点）
- 命中后走 Side Panel 学习流，跳过 Popover

### 2. 语法讲解

- 用户拖选 **单词或短语**（不要求选中整句）→ 点 Popover「学习」或 `Alt+G` → Side Panel 展示句意、成分、从句/时态、难点
- **语境由系统自动补全**（见下文「语境补全」）；引用区展示补全后的原文，**用户选中的词/短语加粗**
- 翻译解决「什么意思」，语法解决「为什么这么说」
- 按需触发，不强迫每次查词都看语法

#### 语境补全（选区 → 讲解语境）

用户阅读时自然选中的是 **词或短语**，不是整句。语法讲解需要句子级上下文，因此由插件在后台补全，不要求用户重新拖选整句。

**两个字段（实现与 API 共用）：**

| 字段 | 含义 |
|------|------|
| `selection` | 用户拖选的原文（词 / 短语 / 句），是学习焦点 |
| `sentence` | 系统推断的讲解语境，供翻译、语法、留标绑定 |

**补全优先级（三级降级）：**

```text
1. 完整句子 — 在页面正文中定位选区，向两侧扩展到最近 . ! ? 边界
   → 成功且长度合理（建议上限约 300 字符）→ sentence = 整句

2. 语境片段 — 无法形成完整句子时（标题、列表项、无句末标点的段落等）
   → 扩展到最近块级容器边界（p / li / td / h1–h6 等）→ sentence = 该块文本

3. 仅选区 — 页面上找不到选区，或块级文本与 selection 相同 / 过短
   → sentence = selection（不再硬扩）
```

**判定流程：**

```text
在正文 innerText 中定位 selection
  ├─ 找不到 → 仅选区（sentence = selection）
  └─ 找到
       ├─ 向 .!? 扩句 → 得到有效整句 → 完整句子
       ├─ 否则扩到最近块级元素 → 文本长于 selection → 语境片段
       └─ 否则 → 仅选区
```

**若用户已选中整句或更长内容：** 当 `selection` 已含句末标点（`. ! ?`）且明显长于单词时，可直接以 `selection` 作为 `sentence`，无需再扩句。

**典型场景：**

| 场景 | 级别 | sentence 来源 |
|------|------|----------------|
| 正文单词 `notwithstanding` | 完整句子 | 自动扩到所在句 |
| 标题 `Contract Law Weekly` | 语境片段 | 标题块全文 |
| 按钮 `Learn more` | 仅选区 | 与 selection 相同 |
| 用户拖选一整句 | 完整句子 | 与 selection 相同或近似 |

**界面反馈（Side Panel 引用区）：**

| 级别 | 引用区 | 辅助说明（可选一行小字） |
|------|--------|--------------------------|
| 完整句子 | 补全句原文 + selection 加粗 | 无 |
| 语境片段 | 块级文本 + selection 加粗 | `基于当前段落讲解（非完整句子）` |
| 仅选区 | 仅 selection | `语境较短，在正文中选中可获得更完整讲解` |

**产品原则：**

1. **永远有可展示内容** — 宁可降级，不空白、不阻断
2. **诚实标注语境质量** — 让用户知道「这句从哪来」
3. **不阻断主流程** — 无完整句子仍可查词、学习、留标；讲解深度可随语境级别调整
4. **焦点始终是 selection** — 无论 sentence 是整句、段落还是单词，高亮与词库均以用户选中内容为准

**与 API：** `POST /v1/translate`、`POST /grammar`、`PUT /v1/words/mark` 均携带 `selection` + `sentence`（及 `page_url` 等）；后端以 `sentence` 为语境、`selection` 为焦点生成讲解与落库。

### 3. 行为留痕（Keep）

- 记录每次 **翻译、语法查看、★ 留标** 及对应语境（句子、页面、出现次数）
- 不是只存单词，而是存 **你在读什么、怎么读的**
- 为后续复习、统计、个性化降噪提供数据基础

### 4. 留标落库（Mark）

- **☆ / ★** 标记你想长期保留的词
- 已标记词持久可查；未标记但反复出现的词 **自动略过**，减少噪音
- 句内词库展示当前句拆分词汇及 **出现次数**

### 5. 阅读面板

- Side Panel：**语法** + **词库**
- Popup：快捷入口与阅读统计
- 快捷键：`Alt+G` 语法 · `Alt+S` 留标 · `Esc` 关闭

## 长期愿景

```text
真实阅读 → 最小交互获取翻译/语法 → 行为被完整记录
         → 词与句在语境中反复相遇 → 理解从「查」变成「懂」
         → 逐渐接近母语者的阅读状态
```

KeepMark 相信：**母语感不是一次背完出来的，而是在足够多、足够真的阅读行为里长出来的。**

---

## 产品边界（v1）

- 不做全页自动翻译
- 不做与阅读无关的独立背单词流程（复习/导出等后续迭代）
- 不承诺「短期速成」，强调 **长期行为积累**

---

## 目标用户

读英文长文、希望 **读得更顺、积累更真、最终少依赖翻译** 的读者与进阶学习者。

---

## 成功标准（产品层）

1. 获取翻译 **无需额外点击**（选中即译）
2. 关键阅读行为 **均可被记录、可追溯**
3. 词库 **随时间更准、更静**（见过的噪音词自动退场）
4. 用户主观感受：读同一类文章时，**查词频率下降、阅读流畅度上升**
