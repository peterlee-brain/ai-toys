# 核心功能（个人自媒体工具）

> **定位**：面向**个人创作者**的一体化自媒体工作台——从灵感记录、内容生产、监听互动到排期发布与复盘，优先覆盖 **X（Twitter）**，再扩展到多平台。  
> **原则**：单人或小团队可维护；先做「能闭环」的最小版本，再叠加智能化与差异化。

---

## 一、需求总览（按能力域分类）

下列功能按「对个人创作者的价值」归类，便于排期；每条附带**实现复杂度**提示（低 / 中 / 高），仅供规划参考。

### 1. 灵感与内容生产

| 功能 | 说明 | 复杂度 |
|------|------|--------|
| **灵感速记 → 成稿** | 随手记下点子，系统结构化（标题、论点、素材槽位），再一键扩写成推文 / Thread 草稿 | 低～中 |
| **个人语气 / 风格** | 基于历史高互动内容做「风格约束」（Prompt + RAG 即可；进阶再上微调/LORA） | 中～高 |
| **长文 → Thread** | 长文本自动拆成多条推文：钩子首条、段落长度、编号与承接句 | 低～中 |
| **推文表现预演** | 发前根据历史数据给出互动区间与改写建议（可先规则化 + 简单模型） | 中 |
| **视觉卡片** | 文字生成适合 X 尺寸的信息图 / 金句图（如 1200×675） | 中 |
| **一稿多形态** | 同一条「核心观点」衍生 Thread、稍正式的短文口径、其它平台口吻（后续再接 LinkedIn / Threads 等） | 中～高 |

### 2. 监听、互动与曝光

| 功能 | 说明 | 复杂度 |
|------|------|--------|
| **账号 / 关键词监听** | 监控指定账号或主题的新动态，聚合到 inbox | 中 |
| **回复草稿 + 蹭曝光** | 对高价值推文生成回复建议；**默认人工确认后发**（合规与账号安全） | 低～中 |
| **热点聚合** | 当日话题榜单、可选领域过滤；与「灵感 → 成稿」联动 | 中 |
| **热点更深一层** | 不只罗列热点：情绪基调、生命周期（早/中/晚期）、是否值得跟 | 高 |
| **精准互动雷达** | 跟踪特定大 V 推文下的新评论，辅助你选择回复时机与话术（仍以人工发送为主） | 中～高 |
| **冷启动互助** | 小圈子约定互动窗口（需严格防刷与真人校验，否则高风险） | 高（产品+风控） |

### 3. 发布与节奏

| 功能 | 说明 | 复杂度 |
|------|------|--------|
| **定时发送** | 内容日历 + 队列；对接官方 API 或导出提醒（视平台政策） | 中 |
| **内容日历** | 看板排期、空档提醒；可与热点日历、纪念日联动 | 中 |
| **「矩阵」内容流** | 给定主题/素材源，批量生成若干条候选推文，人工勾选发布 | 中 |

### 4. 情报与竞品

| 功能 | 说明 | 复杂度 |
|------|------|--------|
| **竞品账号扫描** | 对标账号的发文时段、高互动结构、常蹭话题类型（报表级即可） | 高 |
| **内容资产库 + 检索** | 历史发文自动打标签（话题、格式、表现档位），支持自然语言「找回那条爆款」 | 中～高 |

### 5. 数据与粉丝洞察

| 功能 | 说明 | 复杂度 |
|------|------|--------|
| **粉丝资产仪表盘** | 超级传播者、话题→粉丝质量、活跃热力（依赖数据抓取与分析管线） | 高 |
| **自动化周报** | 周涨粉、最佳推文、互动变化、下周建议；推送邮箱 / 钉钉 / 飞书 | 低～中 |

### 6. 工作流、合规与协作

| 功能 | 说明 | 复杂度 |
|------|------|--------|
| **发送前风控** | 敏感词、事实核查标记、二次确认（涉投资/Web3 等场景尤其重要） | 低～中 |
| **直播 / Space 提纲** | 输入主题生成提纲、开场、过渡与互动话术 | 低～中 |
| **赞助与合规标注** | Sponsored 标记、#ad、合作方 handle、链接追踪 | 中 |
| **Affiliate 植入建议** | 在草稿中提示可挂联盟链接的位置 + 点击归因（后期） | 高 |

### 7. 变现闭环（后期）

| 功能 | 说明 | 复杂度 |
|------|------|--------|
| **赞助内容管理器** | 活动排期、披露模板、效果回流 | 中 |
| **订阅内容策略** | 与平台付费订阅能力配合，管理免费/付费比例（策略为主） | 中～高 |

## 二、与仓库现有方向的对齐（简要）

当前产品文档（`product.md`）中的 **MVP** 侧重：**监控 → 提炼 → AI 草稿 → 人工复制发送**，且不默认机器自动发帖。  
上表中凡涉及「自动回复 / 定时直发 / 互助刷互动」的能力，均建议**默认关闭或强制人工确认**，并在上线前单独评估平台 ToS 与合规。

## 三、个人开发建议路线（分步实施）

下面按**依赖顺序**分成阶段；你可按周或按里程碑推进，每阶段结束应有一个「自己能每天用」的增量。

### 阶段 0：地基（第 1 步）

**目标**：账号与平台连通、能拉到自己账号的基础数据、有一条稳定的本地/云端运行方式。

- 登录与平台授权（如 X OAuth）、Token 刷新与安全存储  
- 最小监控：关注列表 / 指定账号时间线接入  
- 统一配置与环境（`.env` / `config.yaml`）、健康检查  

**验收**：能在控制台或简易 UI 里看到自己或监控对象的最新若干条动态。

---

### 阶段 1：闭环 MVP（第 2 步）

**目标**：「看到内容 → 生成回复/短文草稿 → 复制发送」跑通；不接复杂自动化。

- Inbox / 动态列表 + 单条详情  
- 调用 LLM：摘要、要点、**回复草稿**（多版本可选）  
- 一键复制；可选钉钉/邮件通知（已有规划可沿用）  
- **热点聚合（简版）**：列表 + 链接到成稿入口  

**验收**：你每天能用它在 10 分钟内处理一批监控动态并完成发帖决策。

---

### 阶段 2：灵感与生产力（第 3 步）

**目标**：从「被动回复」扩展到「主动创作」。

- **灵感速记**（标题 + 子弹点即可）→ 扩写成推文 / Thread  
- **长文 → Thread** 拆分  
- **内容日历**：草稿状态（构思 / 待发 / 已发）、简单排期  

**验收**：一周内容里至少一半可从「灵感池」产出，而不只靠回复。

---

### 阶段 3：节奏与素材（第 4 步）

**目标**：稳定产出频率，减轻「不知道发什么」。

- **矩阵式批量候选**：同一主题生成多条候选，人工勾选  
- **定时提醒或队列**（若暂不直连发帖 API，可用日历提醒 + 深链）  
- **周报**：核心指标 + 上周最佳 + 下周 3 条方向建议  

**验收**：你能提前排好一周草稿槽位，并有数据反馈闭环。

---

### 阶段 4：差异化情报（第 5 步）

**目标**：从工具变为「带判断的运营助手」。

- **竞品账号报表**（先做只读统计与样式归纳，不做全自动发帖）  
- **内容资产库**：导入历史推文、标签与检索  
- **热点阶段判断**（规则 + LLM 结构化输出）  

**验收**：你能回答「对标账号最近在测什么叙事」「我该跟哪类热点」而有依据。

---

### 阶段 5：风格深化与视觉（第 6 步）

**目标**：弱化「AI 味」，强化个人品牌。

- 基于历史高互动内容的 **RAG + Prompt 风格包**  
- **视觉卡片**流水线（模板优先，生成次之）  
- **发前预演**（规则版 → 模型版）  

**验收**：陌生人阅读下，语调更接近你的历史内容；带图帖子占比提升

---

### 阶段 6：风控与变现模块（第 7 步）

**目标**：规模化时的安全与商业接口。

- **发送前风控**（敏感词、二次确认、可选事实核查标记）  
- **赞助 / Affiliate / 订阅策略** 的可选模块（与账号阶段匹配再上）  

**验收**：合作内容与常规内容可分区管理，核心合规路径可追溯。

## 四、如何使用本文档

- **写 PRD / issue**：从第一节表格勾选题名，注明所属阶段。  
- **与 AI 协作**：先说明当前处于「阶段 N」，避免一次性生成全局代码。  
- **与 `product.md` 同步**：接口与页面级细节仍以 `product.md` 为准；本文负责**能力地图与个人开发节奏**。
